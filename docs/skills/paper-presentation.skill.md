# 论文分享 PPT 制作

把论文讲给同行听，不是把论文塞进幻灯片。每页只承载一个观点，文字是提示词，细节放进演讲者备注（notes）。

## 模板与字号（硬规范）

- 使用实验室模板版式：标题页 / 目录 / 节标题 / 标题和内容 / 两栏 / 比较 / 图片与标题 / 末尾页。
- 字号固定四档：大标题 30、小标题 20、正文 18、小字（图注/备注）16。不要自造字号。
- 标题一律短标题（≤20 字），说人话：「为什么现有方法会失败」优于「现有方法局限性分析」。
- 全文禁止破折号（— – ——），改用逗号、冒号或拆句。

## 结构套路

- 单篇分享：封面 → 目录 → 背景与动机 → 方法（直觉先行，再讲关键设计）→ 实验与结果 → 局限与启发 → 结束页。10-15 页。
- 多篇梳理：封面 → 目录 → 按主题线分节（每条线：共性问题 → 各家做法对比 → 小结）→ 跨论文总结 → 结束页。禁止逐篇流水账，「比较」版式用于方法对比。
- 每个章节用「节标题」页过渡；目录 ≤5 条。

## 图文并茂

- 论文的重要配图尽量都用上：方法图放方法节、实验图放结果节，一页一图。
- 每张图必须配讲解（caption，16 号字）：这张图画了什么、支撑哪个论点、观众该看图中哪个位置。
- 选图优先架构图和主结果图；图中文字要能看清楚，看不清的表格改成要点复述而不是贴截图。

## 内容密度

- 每页正文 ≤6 条要点，每条 ≤40 字，一行一个短句；讲细节用二级要点（行首「- 」）。
- 数字要带对比锚点：「比 CoT 高 8.4 分」优于「效果显著提升」。
- 演讲者备注写这一页要说的话（口语，2-4 句），不是要点的复读。

## 渲染反馈迭代

生成后必须过两道检查再交付：①规范检查（字号/标题长度/破折号/条数），不过就修；
②渲染成图逐页看：文字溢出、图片变形或过小、图中文字看不清、版面拥挤或大片留白，发现问题回去改内容或换版式，而不是硬塞。
